Code Log 1: HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

Semantic tags tell you what they're meant for in a readable way. For example, nav is where navigation links go, and header is where a header goes. Non-semantic tags are kind of the opposite. Tags like div and span don't say anything about what they're supposed to contain.

What are two (or more) benefits of using semantic tags?

  1. They allow for better readability for the developer(s).
  2. They're more accessible for screen readers.
  3. They're easier to work with, since, being descriptive as they are, it's easier to remember what they do.
